**Schedule action (UI-09):**
Renders inline: `Input type="datetime-local"` with `min` = current datetime ISO (truncated to minute), + "Schedule Campaign" `Button variant="outline"`.
Before POST: `new Date(inputValue).toISOString()` to convert local time to UTC ISO.
`useMutation` â†’ `POST /campaigns/:id/schedule` with `{ scheduled_at: isoString }`.
On success: `invalidateQueries(['campaign', id])` + `invalidateQueries(['campaigns'])`.

**Live polling (UI-10):**
`refetchInterval: (query) => query.state.data?.status === 'sending' ? 2000 : false`
Applied to the `useQuery(['campaign', id])` query.
Polling stops automatically when status transitions to `sent` (interval returns `false`).

**Skeleton states:**
- `isPending` on detail query: full-page `Skeleton` blocks matching section heights
- Specifically: name skeleton `h-8 w-48`, stats skeleton `h-4 w-full` Ă— 2, actions skeleton `h-10 w-24` Ă— 2

**Recipient status indicators:**
Each `campaignRecipient` row renders a `span` with color based on status:
- `pending`: `text-muted-foreground`
- `sent`: `text-green-600`
- `failed`: `text-destructive`

**Logout (UI-13):**
Rendered in page header or nav. `Button variant="ghost" size="sm"`.
`useMutation â†’ POST /auth/logout â†’ onSettled: dispatch(clearAuth()) â†’ navigate('/login', { replace: true })`.
`onSettled` (not `onSuccess`) â€” clears auth even if logout API returns 401 (expired token).
`dispatch(clearAuth())` BEFORE `navigate('/login')` â€” order is mandatory (RESEARCH.md Pitfall 7).

## Critical Implementation Notes

The following are binding constraints captured from RESEARCH.md pitfalls â€” executor MUST follow:

1. **Pagination model (CRITICAL):** `GET /campaigns` is OFFSET pagination. `useInfiniteQuery` uses `initialPageParam: 1` and `getNextPageParam` reads `pagination.page < pagination.totalPages`. There is NO `nextCursor` field.

2. **React Query v5 `refetchInterval` signature (CRITICAL):** `(query) => query.state.data?.status === 'sending' ? 2000 : false`. NOT `(data) => data?.status`.

3. **datetime-local timezone (CRITICAL):** Always `new Date(localDatetimeValue).toISOString()` before POST. Raw datetime-local string fails Zod `z.string().datetime()` validation (no TZ indicator).

4. **stats rates are decimal (0.0â€“1.0):** Multiply by 100 before passing to `Progress value`. `<Progress value={(stats.send_rate ?? 0) * 100} />`.

5. **shadcn add directory:** Run from `frontend/` directory only. Not from repo root.

6. **Logout order:** `dispatch(clearAuth())` BEFORE `navigate('/login')`.

7. **Campaign body in detail page:** Render as plain text (`<p>{campaign.body}</p>`). Never `dangerouslySetInnerHTML` â€” XSS risk.

8. **`useInfiniteQuery` v5:** Must include `initialPageParam: 1` â€” required in v5 (TypeScript error if missing).
