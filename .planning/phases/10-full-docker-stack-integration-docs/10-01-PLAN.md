Create `backend/Dockerfile` with this exact content:

```dockerfile
# backend/Dockerfile — multi-stage build
# Stage 1: compile TypeScript to dist/
FROM node:20-alpine AS builder

WORKDIR /app

# Install corepack so Yarn 4 is available
RUN corepack enable

# Copy workspace manifests and lock file first (layer cache)
COPY package.json yarn.lock .yarnrc.yml ./
COPY shared/package.json ./shared/
COPY backend/package.json ./backend/
COPY frontend/package.json ./frontend/

# Install ALL deps (including devDeps for tsc)
RUN yarn install --immutable

# Build shared first (backend imports from @campaign/shared dist/)
COPY shared/ ./shared/
RUN yarn workspace @campaign/shared build

# Copy backend source and compile
COPY backend/ ./backend/
RUN yarn workspace @campaign/backend build

# Stage 2: production image
FROM node:20-alpine

WORKDIR /app

RUN corepack enable

# Copy workspace manifests and lock file
COPY package.json yarn.lock .yarnrc.yml ./
COPY shared/package.json ./shared/
COPY backend/package.json ./backend/
COPY frontend/package.json ./frontend/

# Install production deps only
RUN yarn workspaces focus @campaign/backend --production

# Copy compiled output from builder
COPY --from=builder /app/backend/dist ./backend/dist

# Copy shared dist (runtime types/validators used at runtime)
COPY --from=builder /app/shared/dist ./shared/dist

# Copy sequelize-cli runtime files (excluded from tsc, needed for db:migrate)
COPY backend/.sequelizerc ./backend/.sequelizerc
COPY backend/src/migrations ./backend/src/migrations
COPY backend/src/seeders ./backend/src/seeders
COPY backend/src/db/config.cjs ./backend/src/db/config.cjs

WORKDIR /app/backend

EXPOSE 3000

# Run migrations then start the server
CMD ["sh", "-c", "yarn sequelize db:migrate && node dist/index.js"]
```

Key design decisions in this Dockerfile:
1. `corepack enable` in both stages — Yarn 4 is required (not classic yarn 1.x)
2. `yarn install --immutable` in builder (fails if lock file is stale — reproducible builds)
3. `yarn workspaces focus @campaign/backend --production` in production stage (installs only @campaign/backend's production deps, not all workspace dev deps)
4. Migrations, seeders, and config.cjs copied explicitly — these are excluded from tsconfig.json's include pattern and are never compiled by tsc
5. CMD uses `sh -c "..."` to allow shell chaining (`&&`)
6. WORKDIR set to /app/backend before CMD so relative paths in .sequelizerc (using __dirname) resolve correctly
