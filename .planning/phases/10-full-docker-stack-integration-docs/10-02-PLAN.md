Create `nginx.conf` at the repo root with this exact content:

```nginx
server {
    listen 80;
    root /usr/share/nginx/html;
    index index.html;

    location /api/ {
        proxy_pass http://api:3000/api/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    location /track/ {
        proxy_pass http://api:3000/track/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

Key design decisions:
1. `proxy_pass http://api:3000/api/` — trailing slash on both sides ensures the path prefix is preserved correctly (nginx strips the `location` prefix and prepends the proxy_pass path)
2. `proxy_http_version 1.1` — required for keep-alive connections to upstream
3. `try_files $uri $uri/ /index.html` — React Router SPA fallback: any URL that isn't a real static file falls back to index.html so client-side routing works
4. No VITE_API_URL, no CORS headers — everything is same-origin from the browser's perspective

Create `frontend/Dockerfile` with this exact content:

```dockerfile
# frontend/Dockerfile — multi-stage build
# Stage 1: build the React SPA
FROM node:20-alpine AS builder

WORKDIR /app

# Install corepack so Yarn 4 is available
RUN corepack enable

# Copy workspace manifests and lock file first (layer cache)
COPY package.json yarn.lock .yarnrc.yml ./
COPY shared/package.json ./shared/
COPY backend/package.json ./backend/
COPY frontend/package.json ./frontend/

# Install all deps
RUN yarn install --immutable

# Build shared first (frontend imports from @campaign/shared dist/)
COPY shared/ ./shared/
RUN yarn workspace @campaign/shared build

# Copy frontend source and build
COPY frontend/ ./frontend/
RUN yarn workspace @campaign/frontend build

# Stage 2: nginx production image
FROM nginx:alpine

# Replace default nginx config with our SPA + proxy config
COPY nginx.conf /etc/nginx/conf.d/default.conf

# Copy compiled SPA from builder stage
COPY --from=builder /app/frontend/dist /usr/share/nginx/html

EXPOSE 80
```

Key design decisions:
1. Build context is the repo root (set in docker-compose.yml) — allows COPY of root manifests and shared/
2. `COPY nginx.conf /etc/nginx/conf.d/default.conf` — replaces nginx's default config
3. No VITE_API_URL env var set during build — frontend makes requests to relative paths only
4. `yarn workspace @campaign/frontend build` runs Vite production build → outputs to frontend/dist/
5. The nginx:alpine stage has no Node.js — minimal final image
