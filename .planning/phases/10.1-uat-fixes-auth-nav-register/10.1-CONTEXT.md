<decisions>
## Implementation Decisions

### Auth Bug Fix (bugs 1–3)

- **D-01:** Fix the axios 401 interceptor in `frontend/src/lib/apiClient.ts` using a **URL allowlist**: if the failing request URL includes `/auth/login`, `/auth/refresh`, or `/auth/register`, skip the retry logic entirely and return `Promise.reject(error)` immediately. This prevents the interceptor from swallowing 401s that belong to the auth endpoints themselves (login failures, expired refresh cookies), which currently causes the button to stay in loading state and hard-redirects the page instead of letting React Query's `onError` and the bootstrap hook handle them.
- **D-02:** After the fix, `loginMutation.isError` will correctly fire on 401 — no additional `onError` handler needed since `LoginPage.tsx` already renders `{loginMutation.isError && <p>...</p>}`.
- **D-03:** `useBootstrap.ts` and `ProtectedRoute.tsx` are correct in logic and require no changes once the interceptor is fixed.

### Navigation

- **D-04:** Add a persistent **top nav bar** component (`frontend/src/components/NavBar.tsx`) rendered only inside protected routes.
- **D-05:** Nav bar layout: app title "Campaign Manager" (left, as a `<Link to="/campaigns">` text), "Campaigns" nav link + Logout button (right).
- **D-06:** Nav bar appears on all protected pages (`/campaigns`, `/campaigns/new`, `/campaigns/:id`) but NOT on `/login` or `/register` (public standalone pages).
- **D-07:** Implement a layout wrapper — wrap protected route content with `<NavBar />` + `<main>` container. Could be a `<ProtectedLayout>` component that wraps NavBar + children, or inline in each ProtectedRoute child. Claude's discretion on exact implementation approach.
- **D-08:** Logout button in NavBar calls `POST /api/auth/logout`, dispatches `clearAuth()`, shows a "You've been logged out" toast via sonner, then navigates to `/login`. No confirmation dialog.

### Register Form

- **D-09:** `RegisterPage` mirrors `LoginPage` visual structure: centered card, same shadcn components (Card, CardContent, CardHeader, CardTitle, Input, Label, Button).
- **D-10:** Show all 3 fields from `RegisterSchema`: name (first), email, password. No confirm-password field.
- **D-11:** On success: `/auth/register` returns `{ accessToken, user }` (same shape as login). Dispatch `setAuth()` and navigate to `/campaigns` — auto-login flow, no intermediate /login redirect.
- **D-12:** Use `RegisterSchema` from `@campaign/shared` with `zodResolver` (same pattern as LoginPage).

### Login↔Register Linking

- **D-13:** Login page: add a centered text link BELOW the Card — "Don't have an account? Register" with the word "Register" as a `<Link to="/register">`.
- **D-14:** Register page: add a centered text link BELOW the Card — "Already have an account? Log in" with "Log in" as a `<Link to="/login">`.

### Claude's Discretion

- Exact ProtectedLayout vs inline NavBar approach
- NavBar styling details (padding, border-bottom, height) — match shadcn/Tailwind conventions already used
- Logout button variant (ghost or outline)

</decisions>
