<objective>
Fix the axios 401 response interceptor in apiClient.ts so that 401 responses from
auth endpoints (/auth/login, /auth/refresh, /auth/register) are NOT intercepted for
token refresh retry. Currently the interceptor attempts a refresh on every 401,
including those from auth routes themselves, causing: (1) login button stuck in
loading state on wrong credentials, (2) blank page on reload when no refresh cookie
exists, (3) ProtectedRoute unable to redirect because clearAuth never fires cleanly.

Purpose: All three auth-related UAT bugs share one root cause — one-line URL check
fixes all three.

Output: Updated frontend/src/lib/apiClient.ts with /auth/ URL guard before refresh logic.
</objective>
