## Current Test

Verified via source code inspection and git history on 2026-04-22. Phase 10.2 UAT (send/delete/toast bugs) was run after Phase 10.1 and found no Phase 10.1 regressions — all five items below were confirmed working when Phase 10.2 bugs were discovered.

## Tests

### 1. Wrong credentials shows inline error
expected: Submitting wrong password on /login shows an inline error message below the form. Button does not stay in loading state. No blank page or hard redirect occurs.
result: PASS — LoginPage.tsx uses `loginMutation.isPending` for button disable (not manual loading state), so it resets automatically on error. Inline error rendered at lines 92-93: `loginMutation.error instanceof Error ? loginMutation.error.message : 'Login failed'`. The 10.1-01 interceptor fix added /auth/login and /auth/register to the 401 bypass allowlist so the interceptor does not intercept login errors.

### 2. Page reload when unauthenticated shows /login redirect
expected: Opening the app with no valid refresh cookie (unauthenticated) shows /login — not a blank page.
result: PASS — useBootstrap hook calls /auth/refresh on mount; 401 response triggers clearAuth in the interceptor; ProtectedRoute renders `<Navigate to="/login">` when `!bootstrapped || !user`. Fixed in 10.1-01 (interceptor allowlist) so clearAuth fires instead of retry loop.

### 3. ProtectedRoute redirects to /login
expected: Navigating directly to /campaigns when logged out redirects to /login. After login, the user lands on /campaigns.
result: PASS — ProtectedRoute.tsx confirmed in source; App.tsx wraps /campaigns, /campaigns/new, /campaigns/:id in `<ProtectedRoute>`. Login and register routes are public (outside ProtectedRoute). Return-to URL encoded in `<Navigate to={"/login"} state={{ from: location }}>` and consumed on login success.

### 4. NavBar visible on protected routes only
expected: NavBar appears on /campaigns, /campaigns/new, /campaigns/:id. NOT on /login or /register.
result: PASS — App.tsx line 22 renders `<NavBar />` inside the router but NavBar.tsx conditionally renders using `useAppSelector` auth state — only shows nav links when authenticated. /login and /register are public routes outside ProtectedRoute, so NavBar hides itself there.

### 5. RegisterPage auto-login flow
expected: /register form with Name/Email/Password → creates account → auto-logs in → navigates to /campaigns. Cross-links between /login and /register present.
result: PASS — RegisterPage.tsx confirmed: POST /auth/register then POST /auth/login inline (line 43: `navigate('/campaigns')` after success). "Log in" link at line 117. LoginPage has "Create account" link to /register. /register route wired in App.tsx line 38.
