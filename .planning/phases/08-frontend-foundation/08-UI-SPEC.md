## Phase 8 Specific: App Shell & Bootstrap States

Phase 8 delivers infrastructure, not pages. The only rendered output is the
bootstrap shell while `/auth/refresh` + `/auth/me` resolves.

### Bootstrap Loading State (UI-03)

- Rendered when `authSlice.bootstrapped === false`
- Full-screen centered container (100dvh)
- shadcn `Skeleton` or a single `Loader2` lucide icon (24px, `animate-spin`)
- No text — avoids flicker/flash of unauthenticated copy
- Background: `bg-background` (Slate white)

### Route Guard Redirect (UI-04)

- `<ProtectedRoute>` wraps all authenticated routes
- Unauthenticated redirect: `navigate('/login', { state: { from: location } })`
- Return-to URL encoded in `location.state.from` (React Router v6 pattern)
- No visible flash — redirect fires after bootstrap completes
  (`bootstrapped === true && !user`)

### Axios Interceptor States (UI-05)

- Silent — no user-visible UI for token refresh
- On persistent auth failure (refresh returns 401): dispatch `clearAuth()`,
  then `navigate('/login')` — no error toast in this phase (toast lands Phase 9)
- `axios.defaults.withCredentials = true` set at module initialization, globally

## Redux Slice Contract (UI-01 / UI-05)

Redux store owns exactly these fields — no server data permitted:

```typescript
interface AuthState {
  accessToken: string | null;   // in-memory only, never persisted
  user: { id: number; email: string } | null;
  bootstrapped: boolean;        // false until /auth/refresh + /auth/me resolves
}
```

All campaigns, recipients, and stats data must live in React Query cache only.
Any Redux slice touching server data is a contract violation.

## Axios Client Contract (UI-05)

```
axios.defaults.baseURL = '/api'         (nginx proxy in prod; dev proxy in vite.config)
axios.defaults.withCredentials = true   (required for httpOnly refresh cookie)
```

Interceptor behavior (enforced in this phase):
- Request interceptor: inject `Authorization: Bearer {accessToken}` if token present
- Response interceptor on 401:
  - If refresh already in-flight: await the memoized promise (not a new call)
  - If not in-flight: create promise, call `POST /auth/refresh`, store in module-scope ref
  - On refresh success: retry original request with new token
  - On refresh failure (401): dispatch `clearAuth()` + redirect to `/login`
- Exactly 1 network call to `/auth/refresh` for N concurrent 401s (C6 guard)
